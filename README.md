# FUTURE_CS_03

## API Security Risk Analysis

### Objective
Analyze the security of a public REST API using Postman and identify common API security risks.

### API Tested
https://jsonplaceholder.typicode.com/users

### Tools Used
- Postman
- Browser
- JSONPlaceholder API
- GitHub

### Findings
- Missing Authentication
- Sensitive Data Exposure
- No Rate Limiting
- Proper Input Validation (404 for invalid requests)

### Repository Contents
- API Security Risk Analysis Report (PDF)
- Testing Screenshots
- README

### Conclusion
The API correctly handles invalid requests but could be improved by implementing authentication, rate limiting, and limiting exposure of sensitive information.
